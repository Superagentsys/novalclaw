## Challenges

### 1. 锁优化问题

**问题**: Tauri Commands 中 AppState 和 MemoryManager 的锁顺序可能导致死锁。

**解决方案**: 在 Story 5-4 中修复了锁顺序问题，确保先释放 AppState 锁再获取 MemoryManager 锁。

**教训**: 跨语言调用时需要特别注意锁的获取顺序。

### 2. 向量搜索性能

**问题**: L3 向量搜索在大数据量时可能较慢。

**当前状态**: Story 5-5 中实现了基础性能监控，但高级优化（SIMD、HNSW 索引）延后到后续迭代。

**建议**: 在数据量增长后需要重新评估性能需求。

### 3. 类型同步

**问题**: Rust 后端和 TypeScript 前端的类型定义需要手动保持同步。

**案例**: Story 5-9 中添加 `MemoryContextInfo` 时需要在多处同步类型定义。

**建议**: 考虑使用代码生成工具自动同步类型定义。

### 4. 测试中的 Portal 问题

**问题**: AlertDialog 等 Portal 组件在 jsdom 测试环境中不正确渲染。

**解决方案**: Story 5-7 中调整测试策略，验证状态变化而非依赖 Portal 渲染。

## Key Learnings

### 1. 使用 `Arc<Mutex<T>>` 共享状态

MemoryManager 在 AgentService 中的集成需要线程安全访问：

```rust
// 正确方式
pub fn set_memory_manager(&mut self, manager: Arc<tokio::sync::Mutex<MemoryManager>>) {
    self.memory_manager = Some(manager);
}
```

### 2. Hook 中避免 Stale Closure

使用 `mountedRef` 和 `hasLoadedRef` 避免闭包陷阱：

```typescript
const mountedRef = useRef(true);
useEffect(() => {
    return () => { mountedRef.current = false; };
}, []);
```

### 3. Tauri 命令注册

所有新增 Tauri 命令必须在 `lib.rs` 中注册才能被前端调用。

### 4. 数据库迁移测试

添加新字段时需要更新迁移测试中的断言：
- 迁移数量断言
- 索引数量断言
- 测试数据初始化

### 5. 前端类型导入

新类型定义后需要在 agent.ts 等相关文件中导入：

```typescript
import { type MemoryContextInfo } from './memory';
```

## Technical Debt

### 1. 性能优化未完成 (Story 5-5 Tasks 3-6)

以下优化任务延后：
- L1 零拷贝优化
- L2 复合索引优化
- L3 SIMD 向量计算
- 并行检索优化

**优先级**: 中
**建议**: 在实际使用中监控性能，按需优化

### 2. 向量索引预加载

L3 向量索引目前未预加载到内存，可能影响首次搜索延迟。

**优先级**: 低
**建议**: 在应用启动时预加载常用向量索引

### 3. 记忆上下文 Token 计数

Story 5-9 中 `totalChars` 可作为 Token 估算，但不够精确。

**优先级**: 低
**建议**: 实现更精确的 Token 计数以更好地控制上下文窗口
